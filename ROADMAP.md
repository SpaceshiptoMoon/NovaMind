# Roadmap

本文档面向公开仓库读者，说明 NovaMind 当前更值得关注的建设方向。

它不是详细执行计划，也不替代 `docs/plans/` 中的历史方案文档。若你想判断项目接下来重点投入什么，而不是追溯某次重构过程，应优先看这里。

## 当前阶段

当前仓库已经具备以下基础能力：

- 知识空间与知识库管理
- 多模态文档处理与索引
- RAG 问答与检索增强
- Deep Research 工作流
- Agent / MCP / 技能广场基础设施
- Docker 部署与前后端分离开发模式

接下来的重点不是再堆更多概念入口，而是把现有能力做稳、做清晰、做可验证。

## 近期重点

### 1. 知识库主链路稳定化

重点方向：

- 继续收敛知识库配置结构与字段边界
- 统一文档处理、切分、索引与任务追踪链路
- 降低历史兼容层对当前实现认知的干扰
- 补齐文档处理与任务状态的回归验证

### 2. 前端真实业务测试补强

重点方向：

- `user store`、路由守卫、认证与 token 刷新链路
- 知识库关键页面与状态流转
- API 封装层的错误处理、重试与权限行为
- 逐步从“最小基线测试”提升为“真实业务流测试”

### 3. 公共仓库可维护性

重点方向：

- 保持根 README、后端 README、文档导航的一致性
- 把正式设计文档与历史过程文档持续分层
- 继续完善贡献、支持、安全和 issue/PR 协作入口

## 中期方向

### 1. 可观测性与运行稳定性

- 更清晰的任务状态可观测能力
- 更系统的失败恢复与重试策略
- 对文档处理、检索和研究链路补充更多运维视角说明

### 2. 评测与效果闭环

- 知识库评测链路增强
- 从测试集、人工复核到结果导出的闭环体验
- 让“效果验证”成为默认能力，而不是附加功能

### 3. Agent / 技能生态收敛

- 明确哪些能力是核心平台能力，哪些是扩展能力
- 降低新技能接入与审核成本
- 让技能市场与实际业务场景之间的映射更清楚

## 不在近期优先级前列的事情

以下事项不是不做，而是不是当前最优先：

- 仅为展示而扩张的新页面或新概念命名
- 在缺少测试和文档支撑时继续加厚兼容层
- 把大量历史计划文档继续扩写成新的“过程性噪音”
