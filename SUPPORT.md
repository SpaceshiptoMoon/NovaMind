# Support

## How to Get Help

For usage questions, setup problems, and troubleshooting:

- Search existing issues first.
- Open a new issue if you have a reproducible bug or a concrete feature request.
- Use the provided issue templates so the context is complete.

## What to Include

When asking for help, include:

- the exact command you ran
- the exact error message
- relevant logs
- your OS, Python, Node.js, Docker, database, and browser versions
- the NovaMind revision or commit SHA you are using

## What Maintainers Can Usually Help With

- installation and local setup
- Docker startup problems
- backend and frontend development workflow
- API behavior that differs from the documented contract
- reproducible bugs in the repository

## What Maintainers May Not Be Able to Support

- custom forks with major private changes
- private infrastructure debugging without reproduction details
- production incidents without logs or environment information
- general AI model quality tuning outside this repository's integration layer

## Security

Do not use public issues for undisclosed vulnerabilities. Follow [SECURITY.md](./SECURITY.md).
