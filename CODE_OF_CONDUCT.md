# Code of Conduct

## Our Standard

We want NovaMind to be a technically serious and respectful open source project.

Examples of behavior that contribute to a constructive community include:

- discussing ideas and code directly, without personal attacks
- assuming good intent while still challenging weak reasoning
- giving actionable technical feedback
- being clear about uncertainty, tradeoffs, and risk
- respecting differences in experience, language, background, and identity

Examples of unacceptable behavior include:

- harassment, intimidation, or discriminatory language
- personal insults, trolling, or bad-faith escalation
- publishing private information without permission
- derailing technical discussion with hostility or repeated abuse
- spam, deliberate disruption, or malicious contributions

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, and other contributions that violate this code of conduct.

Repeated or severe violations may lead to temporary or permanent exclusion from project spaces.

## Reporting

If you experience or observe unacceptable behavior, report it privately through the repository maintainers' contact channel.

For undisclosed security issues, use the process described in [SECURITY.md](./SECURITY.md).

## Scope

This code of conduct applies to repository discussions, issues, pull requests, review comments, and other project-managed community spaces.
